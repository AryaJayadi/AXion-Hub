## Integration Check Results

### Verified Wiring (15/18 — All Major Paths Connected)

- **GatewayProvider store subscriptions**: All 5 feature stores initialized with cleanup (connection, agent, dashboard, activity, mission)
- **Auth protection**: Dual-layer (proxy.ts edge + layout server) covers all dashboard routes
- **Agent store shared state**: Consistently used by roster, chat picker, agent detail shell, dashboard widgets via useAgents() + useAgentStore
- **Cross-feature navigation**: Agent → Chat (Link), Dashboard → Chat (Link), Sessions → Transcript (Link chain), Per-agent Sessions → SlideOver
- **Alert pipeline structure**: DB schema → BullMQ queue → worker → API endpoint → polling bridge → Zustand store → NotificationBell
- **Real-time data flow**: WebSocket → EventBus → 5 store subscription sets → React components
- **Chat entry points**: "New Chat" and "Start Chat" buttons → AgentPickerDialog → resume-or-create → /chat/[agentId]

### Non-Critical Integration Tech Debt (3 items)

**INT-TD-01: Alert queue has no producer (affects MNTR-04)**
- `alertQueue.add()` is never called anywhere in the application
- `conditionMet = false` hardcoded in `workers/alert-worker.ts:55`
- NotificationBell will always show zero unread at runtime
- The full-stack wiring (queue → worker → DB → API → Zustand → UI) is architecturally correct but the trigger is absent
- Fix: Add EventBus handler or cron trigger that evaluates alert rules and enqueues jobs

**INT-TD-02: Task events not in activity feed (affects DASH-06)**
- `EVENT_NAMESPACES` in `event-mapper.ts` subscribes only to `agent.*`, `chat.*`, `exec.*`, `ws.*`
- `task.*` events are consumed by mission store but invisible in Activity Feed and /activity page
- Fix: Add `"task.*"` to `EVENT_NAMESPACES` array and add case to `getEventDisplayInfo()`

**INT-TD-03: No sidebar nav for developer tools (affects ADEV-01, ADEV-02)**
- `/api-docs` and `/docs` have no entries in sidebar `navigationConfig`
- Accessible via direct URL or through docs hub internal links
- Fix: Add "Developer" group to navigation.ts with entries for `/docs` and `/api-docs`

## Tech Debt Summary

### By Category

**Mock Data Layer (~20 items, Phases 3-10):** All feature API hooks return mock data with `// TODO: Replace with gatewayClient calls` comments. This is the single largest category — an intentional design choice to build the complete UI surface first, then wire to real data.

**Alert Evaluation (Phase 5):** Alert rule CRUD UI, worker infrastructure, and notification pipeline are all built, but the trigger and evaluation logic are absent/skeleton.

**Event Feed Coverage (Phase 5+6):** Task events are not mapped into the activity feed event namespace. One-line fix needed.

**Navigation Completeness (Phase 10+11):** Developer tools (/docs, /api-docs) have no sidebar entries.

**Minor Issues:** `totp-setup-card.tsx` verifyTOTP casing mismatch, `agent-memory-browser.tsx` MDEditor type mismatch, filter-bar basic date input, org-switcher create placeholder.

### Total: ~25 items across 10 phases

All items are documented in phase VERIFICATION.md files. None are blockers.
